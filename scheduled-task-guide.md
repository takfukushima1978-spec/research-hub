# Scheduled Task 運用ガイド

## 概要

`daily-research` Scheduled Task が毎日 8:00 JST に自動実行され、
以下の 3 機能を統合的に処理する（プロンプトの詳細はセッションB側で設定）：
- 機能A: 各プロジェクトの ADR 横断確認 → TBP 昇格候補の検出
- 機能B: 外部情報リサーチ（Claude Code + 会計×AI）
- 機能C: 再検討トリガー照合（A と B の結果を接続）

## Tak の日次確認フロー

1. research-log.md のデイリーレポートを確認する
2. 🔴（即座に適用すべき）があれば優先対応
3. TBP 昇格候補があれば、昇格の要否を判断し、
   best-practices-guide スキルの手順に従って TBP を作成
4. 再検討トリガー該当があれば、該当 ADR/TBP を開いて再検討
5. references.md 更新提案があれば、内容を確認して反映

## 自動実行と手動対応の境界

自動で行うこと:
- 各プロジェクトの decisions/ 横断スキャン
- 外部情報源の検索・分類
- 再検討トリガーとの照合
- レポート作成・コミット・プッシュ

手動で行うこと（Tak の判断が必要）:
- TBP の作成・更新
- references.md の更新
- 各プロジェクトのスキル・ルール変更
- 🔴 事項への対応

## 運用の見直し基準

4 週間運用した段階で以下を確認する：
- デイリーレポートは実際に読まれ活用されているか
- 毎日の頻度は適切か（情報が薄い日が多ければ頻度を下げる）
- ADR → TBP 蒸留は機能しているか
- 再検討トリガー照合で実際に再検討が発生しているか
見直し結果は ADR-005 として記録する。
