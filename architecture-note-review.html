01

使い方

architecture noteを実際に使いながら、思いついた良かった点・改善点を5軸に分けて追記していく。各項目には3種類のフラグのうちひとつを付ける。入力内容はブラウザに自動保存され、複数セッションにまたがって追記できる。Exportで Claude Code に渡せる形に書き出せる。

🌱 skill化したい

このHTML固有の話ではなく、他のarchitecture noteでも使えるパターンとして抽出したい要素。

🔧 このHTML固有の修正

今のresearch-hub-v2のnoteだけで起きている問題で、直したいが汎用化はしない。

❓ 判断保留

良し悪しを判断するのに材料が足りない、または時間を置きたい項目。

02

評価軸ダッシュボード

5つの軸を俯瞰する。カードをクリックすると該当軸セクションにジャンプする。

08

skill化に向けた論点

🌱フラグを付けた項目が自動でここに集まる。skill化の議論に持ち込む素材として。

🌱 SKILL CANDIDATES

レビューが進むにつれて、ここに「汎用化したい構造要素」が積み上がっていく。最終的にこの一覧を Claude Code に渡して skill 設計を進める。

まだ🌱フラグが付いた項目はありません。各軸セクションで項目を追加し、🌱フラグを付けてみてください。
09

Export
