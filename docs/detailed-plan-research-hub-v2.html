1. コンセプトと行動モデル

「空気のようなインプット」= 意識せずに良質な情報が蓄積され、必要な時に必要な形で取り出せる状態。

Takの一日の行動モデル

朝 (4:00-7:00) — 自動リサーチがGmail下書きに3本配信。起床後にスマホで流し読み。気になるものはクリップ。

通勤 (7:00-9:00) — ビューアーでクリップ記事を精読。既読フラグON。

仕事中 (9:00-18:00) — 参考になった記事をClaude Code/Coworkから手動登録。「この記事をDBに入れて」の一言で。

夜 (20:00-) — 週次で未読記事のキャッチアップ。階層タグで「セキュリティ系だけ」などフィルタ。

週末 — クリップ記事をNotebookLMに投入 → 音声化してながら聴き。

ALTER TABLE research.articles
  ADD COLUMN IF NOT EXISTS source_type TEXT DEFAULT 'auto_research'
  CHECK (source_type IN (
    'auto_research',    -- 自動リサーチ
    'manual_clip',      -- 手動クリップ（URL共有）
    'manual_note',      -- 手動メモ（テキスト入力）

-- 外部記事用のURLフィールド
ALTER TABLE research.articles
  ADD COLUMN IF NOT EXISTS external_url TEXT;

-- UPDATEのRLSポリシー追加（フラグ切替用）
CREATE POLICY "Allow public update articles"
  ON research.articles FOR UPDATE
  USING (true) WITH CHECK (true);

-- インデックス
CREATE INDEX IF NOT EXISTS idx_articles_is_read
  ON research.articles(is_read);
CREATE INDEX IF NOT EXISTS idx_articles_is_clipped
  ON research.articles(is_clipped);

  ON research.articles(source_type);

2.2 階層タグシステム

-- tags テーブルを階層化
ALTER TABLE research.tags
  ADD COLUMN IF NOT EXISTS parent_id UUID
    REFERENCES research.tags(id),
  ADD COLUMN IF NOT EXISTS level INT DEFAULT 1
    CHECK (level IN (1, 2, 3)),
  ADD COLUMN IF NOT EXISTS label_ja TEXT,
  ADD COLUMN IF NOT EXISTS sort_order INT DEFAULT 0,
  ADD COLUMN IF NOT EXISTS icon TEXT;

-- 階層インデックス
CREATE INDEX IF NOT EXISTS idx_tags_parent
  ON research.tags(parent_id);
CREATE INDEX IF NOT EXISTS idx_tags_level
  ON research.tags(level);

-- 階層タグ取得用ビュー
CREATE OR REPLACE VIEW research.tags_tree AS
SELECT
  t.id, t.name, t.label_ja, t.level,
  t.parent_id, t.sort_order, t.icon,
  p.name as parent_name,
  p.label_ja as parent_label_ja,
  gp.name as grandparent_name,
  gp.label_ja as grandparent_label_ja
FROM research.tags t
LEFT JOIN research.tags p ON p.id = t.parent_id
LEFT JOIN research.tags gp ON gp.id = p.parent_id
ORDER BY
  COALESCE(gp.sort_order, p.sort_order, t.sort_order),
  t.level, t.sort_order;

2.3 フラグ切替用RPC関数

-- 既読/クリップのトグル関数
CREATE OR REPLACE FUNCTION public.toggle_article_flag(
  p_article_id uuid,
  p_flag text  -- 'read' or 'clipped'
)
RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = research, public
AS $$
DECLARE
  v_new_value boolean;
BEGIN
  IF p_flag = 'read' THEN
    UPDATE research.articles
    SET is_read = NOT is_read,
        read_at = CASE WHEN NOT is_read
          THEN now() ELSE NULL END
    WHERE id = p_article_id
    RETURNING is_read INTO v_new_value;
  ELSIF p_flag = 'clipped' THEN
    UPDATE research.articles
    SET is_clipped = NOT is_clipped,
        clipped_at = CASE WHEN NOT is_clipped
          THEN now() ELSE NULL END
    WHERE id = p_article_id
    RETURNING is_clipped INTO v_new_value;
  END IF;

  RETURN jsonb_build_object(
    'article_id', p_article_id,
    'flag', p_flag,
    'value', v_new_value
  );
END;
$$;

GRANT EXECUTE ON FUNCTION
  public.toggle_article_flag TO anon, authenticated;

2.4 手動インプット用RPC関数

-- URLから記事を手動登録
CREATE OR REPLACE FUNCTION public.add_manual_article(
  p_title text,
  p_url text DEFAULT NULL,
  p_body_text text DEFAULT NULL,
  p_summary text DEFAULT NULL,
  p_tag_ids uuid[] DEFAULT '{}'::uuid[],
  p_source_type text DEFAULT 'manual_clip'
)
RETURNS uuid
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = research, public
AS $$
DECLARE
  v_article_id uuid;
  v_slug text;
  v_tag_id uuid;
BEGIN
  -- slug生成
  v_slug := to_char(now(), 'YYYY-MM-DD') || '_manual_'
    || substr(md5(p_title || now()::text), 1, 8);

  -- 記事挿入
  INSERT INTO research.articles (
    title, title_ja, slug, body_text,
    summary, external_url, source_type,
    source_date, status
  ) VALUES (
    p_title, p_title, v_slug, p_body_text,
    COALESCE(p_summary, left(p_body_text, 200)),
    p_url, p_source_type,
    current_date, 'published'
  ) RETURNING id INTO v_article_id;

  -- タグ紐付け
  FOREACH v_tag_id IN ARRAY p_tag_ids LOOP
    INSERT INTO research.article_tags (article_id, tag_id)
    VALUES (v_article_id, v_tag_id)
    ON CONFLICT DO NOTHING;
  END LOOP;

  -- ソースURL登録
  IF p_url IS NOT NULL THEN
    INSERT INTO research.research_sources (
      article_id, url, title,
      domain
    ) VALUES (
      v_article_id, p_url, p_title,
      (regexp_match(p_url,
        'https?://([^/]+)'))[1]
    );
  END IF;

  RETURN v_article_id;
END;
$$;

GRANT EXECUTE ON FUNCTION
  public.add_manual_article TO anon, authenticated;

4. ビューアー（GitHub Pages）改修仕様

4.1 フラグ切替UI

記事一覧の各カードに2つのアイコンボタンを追加:

  • 👁 既読ボタン — タップで既読/未読を切替。既読の記事はカードが薄くなる
  • 📌 クリップボタン — タップでクリップON/OFF。クリップされた記事はアクセントカラーのバッジが付く

タップ時に toggle_article_flag RPC関数を呼び出し、UIを即時更新。

4.2 フィルタバー拡張

既存のカテゴリフィルタに加えて:

  • 状態フィルタ: すべて / 未読のみ / クリップのみ
  • ソースフィルタ: すべて / 自動リサーチ / 手動登録
  • タグフィルタ: L1タグをタップ → L2が展開 → L2タップで絞り込み

4.3 手動登録フォーム

ビューアーのヘッダーに「＋」ボタンを追加。タップするとモーダルが開く:

  • URLフィールド（任意）
  • タイトルフィールド（必須）
  • メモ/本文フィールド（任意）
  • タグ選択（L1 → L2の2段階セレクト）
  • 「登録」ボタン → add_manual_article RPC呼び出し

4.4 階層タグ表示

記事カードにタグをバッジ表示。L1タグは色分け:

  • 📊 会計・税務 → 青系バッジ
  • 🤖 AI・テクノロジー → 緑系バッジ
  • 🔧 ツール → オレンジ系バッジ
  • 🔒 セキュリティ → 赤系バッジ
  • 💼 経営 → 紫系バッジ

5. 手動インプットの3つの方法

方法A: ビューアーの「＋」フォーム

スマホで記事を見つけた時に、URLとタイトルをコピペして即登録。最もカジュアルな方法。

方法B: Claude Code / Coworkから登録

「この記事をリサーチDBに追加して」と自然言語で指示。Claude Codeがadd_manual_article RPCを呼び出す。Supabase MCPが接続されていれば直接、なければPythonスクリプト経由。

💡 例: 「https://example.com/article をリサーチDBに追加して。タグはClaude Code + セキュリティで。」

方法C: ブックマークレット（将来）

ブラウザのブックマークバーに登録するJavaScriptワンライナー。任意のWebページ上で実行すると、そのページのURLとタイトルをSupabaseに自動登録。Phase 3で実装。

6. 「空気のようなインプット」運用設計

6.1 自動収集の拡張性

新しいリサーチトピックの追加は、スケジュールタスクを作成するだけ。research_topicsテーブルでトピックを管理し、カテゴリとタグを紐付ければ自動的にビューアーに反映される。

6.2 タグの育て方

  • L1（大区分）: 5つで固定。追加は年1回程度のレビューで検討
  • L2（中区分）: 月次で見直し。使用頻度が低いものは統合
  • L3（小区分）: 必要に応じて随時追加。使い方に合わせて自然に成長させる

6.3 データ鮮度管理

  • 90日以上前 + 未読 + 未クリップ → 自動アーカイブ
  • クリップされた記事は永久保存（アーカイブ対象外）
  • 手動登録した記事もアーカイブ対象外

6.4 週次ダイジェスト（将来）

週末に自動生成。「今週のクリップ記事」「未読の重要記事」をまとめたダイジェストメールをGmail下書きに配信。
