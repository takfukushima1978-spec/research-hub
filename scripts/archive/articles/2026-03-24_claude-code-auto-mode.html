「Auto Mode」という名の権限管理革命
93%の承認率がもたらす生産性向上

2026年3月24-25日 発表

Anthropicは2026年3月24-25日、Claude Code向けの革新的な権限管理機能「Auto Mode」を研究プレビューとしてリリースした。AIモデルが権限要求を自律的に判断し、人間の介入を最小化する仕組みだ。ユーザーの承認率93%という驚異的な数字の背景には、入力層と出力層の二層防御システムと、極めて精密な分類器がある。

1

権限疲弊からの解放
「承認地獄」を終わらせる

Claude Codeの最大の課題は、あらゆる操作に対して人間の承認が必要だったことだ。ファイル作成、コマンド実行、外部APIへのアクセス——一つ一つに「本当に実行していいですか？」と問われる。これは安全性の観点では理想的だが、実務では無視できない負荷になる。

Auto Modeは、この権限疲弊を根本から解決する。モデルベースの分類器が権限要求を自動判断し、真に危険な操作だけを人間にエスカレートする。その結果、93%の操作が承認されることになった。

93%の承認率がもたらすものは、単なる利便性ではない。開発の流速、自動化の深さ、AI時代の生産性基準そのものが変わる。

ユーザーが決断を迫られるのは、実際に危険性が高い操作のみ。つまり「全て信用する」のでもなく「全て疑う」のでもなく、その中間地点——Anthropicが「インテリジェント・ミドルグラウンド」と呼ぶ領域を実現した。

2

二層防御システム
入力層と出力層で悪意を遮断

Auto Modeの核は「二層防御」という戦略にある。第一層は入力側、第二層は出力側。それぞれ独立した防御メカニズムが動作する。

入力層：Prompt Injection Probe（サーバー側スキャナー）
ユーザーのプロンプトをサーバー側で事前スキャンし、明らかなプロンプトインジェクション攻撃を検出する。これはモデルの推論前に機能するため、悪意ある指示がそもそも正当な入力として扱われない。

出力層：Transcript Classifier（Sonnet 4.6 ベース）
モデルの実行ログ（トランスクリプト）を「実行してもいいか」を判定する専用分類器が分析する。ここが Auto Mode の最大の革新点だ。

入力 → 処理 → 出力の全段階で防御が機能する。プロンプトインジェクションと実行時の意図的な不正操作、両方に対応できる設計。

この二層構造により、一つの層をすり抜けた悪意にも、もう一つの層が対抗できる。セキュリティの多層防御原則をAI時代に実装した格好だ。

3

三段階の権限ティア
操作の危険度に応じた段階的許可

Auto Mode の権限判定は、操作の種類によって三つのティアに分かれている。これにより、「すべて許可」と「すべて拒否」の二者択一ではなく、きめ細かい制御が実現された。

Tier 1：Safe-Tool Allowlist（自動承認）
既知の安全なツール（リスト内のコマンドやAPI呼び出し）は自動的に許可される。セットアップ、カレンダー操作、標準ライブラリの読み込みなど、悪用の余地がない操作が対象。

Tier 2：In-Project File Operations（限定的な許可）
プロジェクト内のファイル操作（作成、編集、削除）は制限されたスコープで許可される。ただしプロジェクト外へのアクセスや外部への流出は防止される。

Tier 3：Classifier Review（分類器が判定）
それ以外の操作——外部APIへのアクセス、システム設定の変更、機密ファイルの読み込みなど——は分類器が詳細に分析する。多くは自動許可されるが、危険性が高いと判定されれば人間にエスカレートされる。

操作の危険度に応じた三段階ティアにより、人間の審査負荷を最小化しながら、セキュリティを維持する仕組みが完成した。

4

精度指標が語る真実
0.4%の誤認可率が示す信頼性

Auto Mode の信頼性は、詳細なパフォーマンスメトリクスに基づいている。Anthropic が公開した数字を見ると、この仕組みがいかに精密か、その輪郭が見える。

誤認可率（False Positive）：0.4%（全パイプライン）
実際には危険な操作を安全だと誤判定してしまう割合。0.4%という数字は、10000回の操作のうち40回だけが誤判定されるということ。この精度は、医療診断や航空機の安全システムレベルの要求に匹敵する。

誤拒否率（False Negative）
本来許可されるべき操作を拒否してしまう割合。実際のユーザー操作では17%、合成的なエクスフィルトレーション試験では5.7%。ユーザー体験を損なわない程度に拒否をコントロールしている。

二段階フィルターが機能している：Stage 1は高速な単一トークン判定、Stage 2はフラグされた操作に対するChain-of-Thought推論。

この指標群は、安全性と利便性のバランスがいかに綿密に調整されたかを物語っている。

5

拒否の運用ロジック
連続拒否で初めて人間が介入

Auto Mode には「Deny-and-Continue」という独特のメカニズムが組み込まれている。つまり、分類器が操作を拒否してもプロセスは続行され、人間への自動エスカレーションは条件付きだ。

具体的には、3回の連続拒否または20回の累積拒否に達して初めて、人間の審査に自動的に引き上げられる。これにより、たった一つの疑わしい操作で流れが止まることがない。一方で、パターン的な危険性は検出される。

複数エージェントが同時に動作する環境では特に有効だ。一つのエージェントが「疑わしい」と判定した操作でも、他のエージェントが継続的に同じパターンで拒否されることで、初めて人間に知らされる。これはマルチエージェントシステムの現実的なニーズに応えた設計と言える。

6

有効化の方法
Shift+TabからJSONまで

Auto Mode を有効にする方法は驚くほどシンプルだ。ユーザーには複数の選択肢が用意されている。

リアルタイムトグル：Shift+Tab
Claude Code の UI 上で Shift+Tab を押すと、権限モード（Manual → Auto → Dangerously Skip）がサイクルして切り替わる。試験的に試してみたい、あるいはセッションごとに変更したいユーザーにはこれが最も簡単。

設定ファイル：settings.json
恒久的に Auto Mode を有効にしたい場合は、Claude Code の設定ファイル（settings.json）に defaultMode: "auto" と記述する。これで新規セッションは自動的に Auto Mode で起動する。

UI からワンキーで、JSON 設定ファイルからデフォルト指定——どちらの選択肢も用意されている点が、Anthropic の設計思想を表している。

利用可能なモデルは Sonnet 4.6 と Opus 4.6。初期段階では Team プラン向けにロールアウトされ、その後 Enterprise・API にも拡大される予定。

編集部コメント

Auto Mode の登場は、「AI に何をやらせるか」という問いに新しい答えを与えた。従来は「全信頼」か「全不信」の二者択一だったが、Auto Mode は「段階的信頼」という第三の道を示した。0.4%の誤認可率、93%の承認率——これらの数字は、単なる技術仕様ではなく、AI時代における人間とマシンの新しい協働モデルの到来を象徴している。企業がセキュリティと生産性の両立に困窮する中で、この「インテリジェント・ミドルグラウンド」がどこまで実務的価値を発揮するか。注視する価値は充分にある。
