ターミナルにcronが来た！
/loop コマンドの衝撃

2026年3月7日 ｜ v2.1.71 /loop & cron scheduling
Claude Code v2.1.71で追加された/loopコマンドが開発者の間で話題だ。任意のプロンプトを指定間隔で繰り返し実行できる、いわば「ターミナル内蔵のcronジョブ」。デプロイ監視からログ解析まで、自動化の幅が一気に広がった。
1

/loop ── 定期実行の新スタンダード

使い方はシンプル。/loop 5m check the deployと打てば、5分ごとにデプロイ状況をチェックしてくれる。外部ツールなしで、Claude Codeのセッション内で完結する手軽さが魅力だ。

💡 活用例: /loop 10m check if CI is green on main（10分ごとにCI状態確認）、/loop 30m summarize new error logs（30分ごとにエラーログ要約）

Voice Mode多言語対応

同バージョンでVoice Modeに10言語が追加され、対応言語が合計20に。日本語での音声操作もより自然になった。voice:pushToTalkのキーバインドもカスタマイズ可能に。

📝 編集部コメント

/loopは地味に見えて破壊力のある機能だ。開発中にターミナルを離れることなく、監視・確認系のタスクを自動化できる。「AIがずっと見張ってくれている」という安心感は、開発体験を質的に変える。
