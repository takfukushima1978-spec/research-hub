CLAUDE CODE 公式発表

MCPが「対話」する時代へ
Elicitation機能の登場

2026年3月14日 ｜ v2.1.76 MCP Elicitation
Claude Code v2.1.76でMCPに「Elicitation（対話的情報収集）」機能が追加された。MCPサーバーがユーザーに直接質問を投げかけ、インタラクティブなダイアログで情報を収集できるようになった。同時にモノレポ向けのsparse-checkout対応も。
1

MCP Elicitation ── ツールが質問してくる

これまでMCPツールは「呼ばれたら結果を返す」一方通行だった。Elicitationにより、MCPサーバーがユーザーに対してインタラクティブなダイアログを表示し、追加情報を求めることができるようになった。

たとえばデプロイツールが「本番環境にデプロイしますか？ステージングですか？」と確認を求めたり、DBマイグレーションツールが「どのテーブルを対象にしますか？」とリストを提示したりできる。

💡 ElicitationとElicitationResultのHookイベントも追加され、対話フローをHooksでカスタマイズすることも可能。
📎 Claude Code Changelog
2

モノレポのsparse-checkout対応

worktree.sparsePaths設定により、巨大なモノレポでもworktree作成時に必要なディレクトリだけを取得できるようになった。Agent Teamsで複数のworktreeを作成する際のディスク使用量とクローン時間を大幅に削減する。

📎 Claude Code Changelog

📝 編集部コメント

MCP Elicitationは、AIエージェントと外部ツールの関係を「一方通行」から「対話」に変えるブレークスルーだ。MCPサーバーの開発者にとっては、ユーザー体験を大幅に向上させる武器になる。MCP エコシステムの成熟を感じさせるアップデートだ。
