CLAUDE CODE 公式発表

CLI版Computer Useが降臨！
Week 14アップデート全解剖

2026年4月3日 ｜ Claude Code v2.1.86〜v2.1.91
Claude Codeが矢継ぎ早にアップデートを繰り出している。2026年第14週（3月30日〜4月3日）のリリースでは、CLI版Computer Use、インタラクティブ学習機能、描画の改善など、開発者の日常を確実に変える機能が5つ追加された。一つずつ見ていこう。
1

CLI版Computer Use ── ターミナルからデスクトップを操る

これまでAPI経由でしか使えなかったComputer Use機能が、ついにCLIから直接利用可能になった。ターミナル上でClaude Codeに「このボタンをクリックして」と指示すれば、デスクトップアプリケーションを操作できる。

E2Eテストの自動化や、GUIしか持たないレガシーツールの操作自動化に大きな可能性がある。

💡 ブラウザテスト、デスクトップアプリの自動操作、スクリーンショットベースのデバッグなど、活用シーンは幅広い。
📎 公式発表を読む（Week 14 Digest）
2

/powerup ── インタラクティブにスキルアップ

新コマンド/powerupは、Claude Codeの機能をインタラクティブなレッスン形式で学べる機能だ。「Hooksって何？」「MCPサーバーの設定方法は？」といった疑問に、ハンズオン形式で答えてくれる。

チームへのオンボーディングツールとしても有用。新メンバーにClaude Codeの使い方を教える手間が大幅に減る。

💡 ターミナルで/powerupと打つだけ。知らなかった機能に出会える発見感が楽しい。
📎 公式発表を読む（Week 14 Digest）
3

フリッカーフリーレンダリング ── 目に優しいUI改善

地味だが確実にうれしいアップデート。ターミナルでの出力がちらつかなくなった。長時間の作業で目が疲れる開発者にとって、この改善は日々の体験を底上げする。

📎 公式Changelog
4

MCP result-sizeオーバーライド

MCPサーバーからの応答サイズ制限をプロジェクト単位でカスタマイズできるようになった。大量のデータを返すMCPツールを使う場合に、デフォルトの制限に引っかからずに済む。

📎 公式発表を読む（Week 14 Digest）
5

プラグイン実行ファイルのPATH追加

プラグインが提供する実行ファイルが自動的にPATHに追加されるようになった。これにより、プラグインのCLIツールをフルパス指定なしで呼び出せる。小さな改善だが、日々の開発フローがスムーズになる。

📎 公式Changelog

📝 編集部コメント

Week 14の目玉はなんといってもCLI版Computer Use。これまで「コードは書けるけどGUIは触れない」という制約があったClaude Codeが、いよいよデスクトップ全体を操作できるようになった。/powerupもユニークで、AIツール自身が使い方を教えてくれるという体験は新鮮。アップデート頻度の高さが、Anthropicのこの領域への本気度を物語っている。
