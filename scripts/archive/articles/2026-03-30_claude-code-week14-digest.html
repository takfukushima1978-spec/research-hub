Claude Code News Digest

Week 14: 進化が止まらないClaude Code

2026年3月30日 - 4月3日 | v2.1.86 - v2.1.91

Computer Useに注目が集まるClaude Code 2.1ですが、その陰では着実な改善が続いている。Week 14は6つのマイナーバージョンアップを重ね、ユーザー体験の細部を磨き上げた。インタラクティブ学習から描画パフォーマンスの革新まで、今週のアップデートから目を離せません。

1

Week 14の概要：機能の進化が加速

3月30日から4月3日にかけて、Claude Codeは6回のバージョンアップ（v2.1.86 ~ v2.1.91）を実施しました。Computer Useの話題に隠れがちですが、開発者体験の向上に直結する改善が集中しているのが特徴です。

特に注目すべきは、複数の改善が「機能の発見と利用」という共通テーマで結びついている点。新機能の存在を知らないユーザーが多いという課題に対し、Anthropicは学習ツール、パフォーマンス最適化、プラグイン連携といった複数のアプローチで対処しています。

Computer Useには専門の記事が設けられため、本稿ではそれ以外の重要な改善に焦点を当てます。
2

/powerup：ターミナルで学ぶインタラクティブレッスン

v2.1.90で導入された/powerupは、Claude Codeの隠れた機能をユーザーに教えるための革新的なツールです。短いアニメーションデモを使い、機能ごとのビジュアルチュートリアルをターミナル上で直接実行できます。

「Claude Codeのリリースは頻繁で、多くの機能がユーザーの目に触れないまま過ぎ去ってしまう」という開発チームの認識が背景にあります。/powerupを一度実行すれば、その時点での新機能が何かが一目瞭然。アニメーションデモにより、テキストだけでは伝わりにくい動作をリアルタイムで体験できるのが強みです。

これはドキュメント充実の次のステップ。「機能が存在すること」から「機能が何をするか直感的に理解できること」へ、UXが一段階進化したと言えます。

3

フリッカーフリーレンダリング：描画の根本改善

v2.1.89で実装された「新型alt-screenレンダラー」は、Claude Codeの応答性とビジュアル品質に根本的な改善をもたらします。従来の描画では、長い会話履歴が蓄積されると、画面の再描画時にちらつき（フリッカー）が発生していました。

新レンダラーの特徴は三点。第一に「仮想スクロールバック」により、実際に見える範囲のみを効率的に描画します。第二に「プロンプト固定機能」で、入力欄が常に画面下部に配置されたまま。第三に「マウス選択機能」が長い会話全体に対応し、任意の範囲をコピーできるようになりました。

環境変数CLAUDE_CODE_NO_FLICKER=1を設定することで有効化可能。特に長時間の作業セッションで威力を発揮します。

これは単なる「見た目の改善」ではなく、ユーザーが集中して作業を続けられる環境設計です。ちらつきのストレスがなくなると、認知負荷が大幅に軽減されます。

4

MCP結果サイズオーバーライド：大規模ペイロード対応

MCPサーバー開発者向けの重要な改善がv2.1.91に実装されました。これまで、MCPツールが返す結果のサイズは全サーバー共通の上限に制限されていました。大規模データを扱うツールでは、結果の一部がディスクに保存される事態が発生していたのです。

新機能により、MCPサーバー作者はanthropic/maxResultSizeCharsパラメータをツール定義に追加し、ツール個別の結果サイズ上限を設定できるようになりました。上限は最大50万文字。DBクエリの結果取得、大規模ファイル解析、複数リソースの一括処理といった、データ量が多いツールの実装がより効率的になります。

従来は全ツール共通の「グローバル制限」で一律カット。今後は「ツール単位でのきめ細かい調整」が可能に。

MCPエコシステム全体の成熟を示す改善です。ツール開発の自由度が高まることで、より実用的で多機能なMCPサーバーが登場することが期待されます。

5

プラグイン実行ファイルのPATH対応

v2.1.91では、プラグイン開発者にとって重要な新しい仕組みが導入されました。プラグインのルートディレクトリにbin/フォルダを配置し、その中に実行ファイルを置くと、Claude Codeがプラグイン有効化時に自動的にこれらをBashツールのPATH環境変数に追加するようになったのです。

この変更の意義は何か。従来、プラグインで独自コマンドを提供する場合、複雑なセットアップが必要でした。今後は、ユーザーがプラグインを有効化するだけで、そのプラグイン固有のコマンドラインツールが即座に利用可能になります。

例えば、Pythonスクリプト実行、データ処理ツール、カスタムビルドスクリプトといったプラグイン要素が、プラグイン有効期間中は自動的にシェル上で実行できる。これによりプラグインの実用性が飛躍的に向上します。

6

その他の注目改善：細部が光る設計

Week 14はこれまで紹介した大型機能だけでなく、ユーザー体験を着実に改善する細かい調整が多数含まれています。いくつか注目すべき改善を紹介します。

PermissionDenied フック対応
自動実行モード（auto mode）において、権限不足で処理が失敗した際に自動的に回復するためのフックが実装されました。プラグインやカスタム設定がより堅牢に動作するようになります。

PreToolUse フックの「defer」値
hooks/PreToolUseにおいて、permissionDecisionに「defer」値が新たに設定可能になりました。これにより、権限判定を後延ばしにし、より柔軟な実行フローが実現します。

その他の実装改善
- disableSkillShellExecution 設定の追加：スキル実行時のシェル呼び出しを禁止
- Edit ツールがcat/sed で表示されたファイルに対応
- フック出力が50K超過時にディスク保存される仕組みの整備
- Thinking モードのサマリー機能がデフォルト無効化
- Voice mode の改善（音声入力品質向上）
- claude-cli:// ディープリンクが複数行プロンプトに対応
- 4月1日限定：かわいい /buddy 生物が登場

これらは一見すると「細かい改善」ですが、重ねると開発者体験の質が大きく向上します。

編集部コメント

Week 14は、Claude Codeが「機能の質」から「体験の質」への転換点を迎えたことを示しています。Computer Useという大型機能が注目を集める一方で、開発チームは地道にUXを磨き、ツール作者向けの基盤を強化しています。/powerupのようなビジュアル学習ツール、フリッカーフリーレンダリングによる集中環境、MCPエコシステムの拡張、プラグイン連携の簡素化—これらが組み合わさると、Claude Codeはやがて「日常的に触るツール」から「手放せない相棒」へ進化していくでしょう。
