Week 13 の概要
怒涛のアップデートラッシュ

わずか5日間で3つのマイナーバージョンアップを敢行した Claude Code。v2.1.83 から v2.1.85 にかけて、単なるバグ修正ではなく、開発ワークフロー全体を変えうる機能が投入された。

世間の注目は Auto Mode と Computer Use に集中しているが、実際のユーザーがもっとも恩恵を受けるのは、むしろキーボードショートカット、OS固有機能、設定管理の柔軟化といった「地味だが必須」の機能たちだ。

Week 13 は「開発者向けOSネイティブ対応」と「UI/UX の細部最適化」に特化したリリースサイクルと位置づけられる。

特に Windows 開発者、自動化パワーユーザー、ターミナル職人にとっては、この week のアップデートは無視できない。

2

Transcript Search
会話検索で迷子から卒業

長い会話履歴から「あのコマンド、あの質問」を探すことは、Claude Code ユーザーにとって日常茶飯事だった。その課題に正面から取り組んだのが Transcript Search である。

実装は極めてシンプルながら強力。トランスクリプトモード中に「/」キーを押すだけで検索モードに突入。n キーで次の一致へ、Shift+N で前の一致へステップスルーできる。Ctrl+O でトランスクリプト自体を開く機能も並行実装された。

「/」→「キーワード」→「n」。この3ステップで、数百行のトランスクリプトから目的の会話に一気にジャンプ。テンポの良さが開発体験を変える。

Git の commit message 検索や、API 呼び出しの試行錯誤ログを遡る際、この機能の価値は格段に高い。ユーザーからの要望も多かったであろう改善だが、週13での実装は遅れて当然とも言える。

PowerShell Tool (Preview)
Windows 開発者への朗報

Mac と Linux で Bash を使う開発者には「Claude Code にシェル機能がある」ことは常識だった。しかし Windows? PowerShell は？という声は、長きに渡って宙ぶらりんだった。

v2.1.84 で導入された PowerShell Tool（プレビュー）は、その空白を埋める。Windows ネイティブの PowerShell コマンドレットを直接実行でき、パイプで複雑なオブジェクト処理も可能。Windows パスの直感的な取り扱いも特徴だ。

opt-in オプション「CLAUDE_CODE_USE_POWERSHELL_TOOL=1」で有効化。環境変数ひとつで、Windows 開発ワークフロー全体が変わる。

現在はプレビュー段階だが、完成度は高い。Active Directory クエリ、Windows Update 管理、Registry 操作といった「Mac/Linux では不可能」なタスクも Claude Code に託せるようになる。企業内での Claude Code 導入を検討していた Windows ショップにとっては、この実装は game changer である。

Conditional Hooks
フック実行を賢くコントロール

Hook 機能自体は既存だったが、v2.1.85 で実装された Conditional Hooks は、その応用可能性を劇的に拡張する。

「すべてのコマンド実行時に Hook を発火させたい」のではなく、「特定の条件下でのみ Hook を発火させたい」という実運用ニーズに応えたのがこの機能だ。

Hook の JSON 定義に "if" フィールドを追記。例えば "if": "Bash(git commit *)" なら、git commit 実行時のみ Hook が動作する。

これにより、「lint 実行前に自動フォーマット」「commit 前に test 実行」といった複雑なワークフロー自動化が、Permission Rule 構文で宣言的に記述できるようになった。Hook の粗粒度な実行制御に悩まされていた power user にとっては、朗報以外の何物でもない。

その他の改善
細部の積み重ねが全体を変える

Week 13 の価値は、上記4つの主要機能だけに留まらない。いくつかの細かいながら実用的な改善が、全体的な開発体験を底上げしている。

Plugin userConfig の公開化： Plugin 開発者向けに userConfig が public になり、keychain ベースのシークレット管理が可能に。プラグインエコシステムの拡張可能性が一段と高まった。

画像チップの自動挿入： ペーストした画像が「[Image #1]」といった chip として自動的に会話に組み込まれるようになった。スクリーンショット共有の煩雑さが軽減される。

managed-settings.d/ ドロップインディレクトリ： 設定管理に段階的な導入が可能になった。個別の設定ファイルを drop-in できるため、複数環境での設定共有が容易に。

CwdChanged / FileChanged などの Hook イベント追加も地味だが重要。ファイルシステムの変化に即座に反応する automation が実現できるようになった。

さらに、Agents が initialPrompt を宣言できるようになり、エージェント型 automation の初期化が柔軟に。Ctrl+X Ctrl+E で外部エディタ起動、/status が Claude の応答中でも利用可能、deep links で preferred terminal を開く、75分以上の idle 後に /clear を nudge するなど、UX の細部が丹念に改善されている。

VS Code ユーザーには rate limit banner の表示、Esc キー 2 回連続で rewind picker が開くといった QOL 向上も実装された。

編集部コメント

Auto Mode と Computer Use が world のニュースを独占する中、Week 13 は「開発者の足元を見つめたリリース」という印象が強い。PowerShell ツール、Conditional Hooks、Transcript Search といった機能は、showroom piece ではなく、daily driver の品質向上に直結している。

特に注目すべきは、OS ネイティブ機能への対応（PowerShell）と、自動化のきめ細かい制御（Conditional Hooks）の実装である。これらは、Claude Code が「単なるチャットインターフェース」から「本格的な開発環境」へと進化していることを示唆している。

細部を大切にする企業哲学が透けて見える week である。
