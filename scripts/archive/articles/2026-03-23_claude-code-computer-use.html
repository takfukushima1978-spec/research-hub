Claude Code Computer Use：AIがデスクトップを操作する新時代

2026年3月23日（Desktop版）/ 3月30日（CLI拡張）

Anthropicが革新的な新機能「Computer Use」を発表した。AIがマウスをクリックし、キーボードを操作し、画面に映る変化を認識する。会話型のAIから「自律型デジタルワーカー」へ。3月23日のDesktop版に続き、3月30日にはCLIでも利用可能になり、ターミナルからGUIアプリを操作できるようになった。

1

Computer Useとは何か
AIがデスクトップを操作する新時代

従来、AIは文字で指示を受け、文字で回答する——その使い方が基本だった。しかしClaudeの新しい「Computer Use」機能は違う。AIが人間と同じように、マウスを動かしてクリックし、キーボードでテキストを入力し、画面に映る変化をリアルタイムで認識する。

この機能によって、Claudeは単なる「回答者」から「実行者」へと進化した。APIが用意されていない古いツール、GUIベースのプロプライエタリアプリケーション、あるいはiOSシミュレータのようにUIを操作する必要のあるすべてのタスクに対応できるようになったのだ。

「APIなしのツールに最適。iOSシミュレータの操作、ハードウェアコントロールパネルの設定変更、GUIオンリーアプリの自動化。これまでAIでは手が出せなかった領域が開かれた。」

デフォルトではオフになっており、Claudeが新しいアプリにアクセスする際には必ず許可を求める。セキュリティは設計段階から組み込まれている。

Desktop版での導入
3月23日、Pro・Max向けに開放

2026年3月23日、Anthropicは「Claude Desktop app」にComputer Use機能を搭載した。対象はClaudeの有料ユーザー（ProおよびMax）である。このとき点灯したのが「研究プレビュー」というステータスだ。つまり、実用段階には至っていないが、一般ユーザーにも試験利用が可能になったということである。

設定方法は簡潔だ。まずClaudeのSettings（設定）を開き、Computer Use機能を有効化する。次に、OSのプライベートシステムアクセス権限をClaudeに付与する。これでセットアップは完了。後はClaudeに任意のタスク実行を依頼し、画面の変化を確認させるだけで良い。

macOS版が最初にリリースされ、Windows対応は今後の計画に含まれている。

「設定 → Computer Useを有効化 → OSパーミッション付与 → Claudeに画面操作を依頼 → エンドツーエンド検証」

CLI版への拡張
3月30日、ターミナルからGUIを操作

わずか1週間後の3月30日（第14週）、Anthropicはこの機能をCLI環境にも展開した。これにより、ターミナルからClaudeに指示を出し、デスクトップアプリの操作を自動化できるようになった。

ワークフローは極めてシンプルだ。ターミナルで/mcpコマンドを実行し、computer-useを探して有効化するだけ。以降、CLIからClaudeに任意のGUI操作を依頼できる。

ターミナル派のプログラマーやシステム管理者にとって、これは革命的だ。スクリプト環境からネイティブアプリを操作し、変化をターミナルで即座に検証できるようになったのである。

「CLIで `/mcp` → `computer-use` をトグルON → ターミナルからGUIアプリを制御可能に」

ユースケース
APIでは実現できなかった自動化

Computer Use機能の真価は、「APIでカバーできていないツール」の領域にある。具体例をいくつか挙げよう。

iOSシミュレータの操作：アプリ開発者にとって、Claudeが直接iOSシミュレータを操作し、タップやスワイプをシミュレートできるのは強力だ。デバイステスト自体をAIに任せられる。

プロプライエタリツール：社内でのみ使われている独自開発ツール、古いレガシーシステム。こうしたツールはAPIが存在しないか、極めて限定的な場合が多い。Claudeが直接UIを操作することで、初めて自動化が可能になる。

GUIオンリーアプリ：ハードウェアコントロールパネル、デザインツール、複雑な設定画面。こうした複雑なUIを持つアプリケーションも、Computer Useなら対応できる。

複雑な検証タスク：画面に映る変化を逐一確認しながら進める必要のあるタスク。たとえば、データベース更新後の画面確認、ビジュアル要素の検証なども、Claudeが目視で確認しながら実行できるようになった。

「これまで『できない』と諦めていた自動化が、今なら『やってみよう』になる」

安全性の設計思想
permission-first、trust-but-verify

強力な機能には、同等の責任が伴う。Anthropicはこの点を深く認識し、Computer Use機能にはセキュリティを徹底的に組み込んだ。

デフォルトはオフ。すべてのユーザーで初期状態は無効化されており、自分で明示的に有効化する必要がある。

アクセス許可を毎回確認。Claudeが新しいアプリケーション、新しいディレクトリへのアクセスを試みるたびに、ユーザーに確認を求める。「信頼するが検証する」というアプローチだ。

OSレベルの権限管理。Settings画面で有効化した後、OSのシステムプライベートアクセス権限を明示的に付与する必要がある。これにより、OSレベルでのアクセス制御が機能する。

透明性と監査。Claudeが何をしたのか、どの画面を確認したのかが、すべてユーザーに可視化される。ブラックボックス化しない設計になっている。

「デフォルトオフ → 明示的な有効化 → 新規アプリアクセス時は確認 → OS権限管理 → 完全な可視化」

Vercept買収の影響と今後の展望
AIの「手足」を手に入れたAnthropics

Computer Use機能が急速に実現できた背景には、2026年2月のAnthropicによる戦略的買収がある。Computer Use技術のスタートアップ「Vercept」の買収だ。

Vercept創業者らがもたらした知見とエンジニアリングが、このComputer Use機能の急速な開発を可能にした。Desktop版が3月23日に、CLIが3月30日に——わずか1週間での拡張という素早さは、統合の成功を物語っている。

今後の展開：現在macOS優先だが、Windows対応は近い。さらに、Linux環境への対応も視野にあるはずだ。APIの一層の拡張、セキュリティ機能のさらなる強化、ユースケースの多様化も予想される。

Anthropicにとって、このComputer Use機能は単なる「新機能」ではない。Claude自体をGUIの世界に解放し、デジタル世界のすべてのツール・アプリに対応可能な「汎用AI」へと進化させるための基盤なのだ。

「Vercept買収 → Desktop版 → CLI拡張 → Windows・Linux対応予定。Claudeが真の『自律型デジタルワーカー』への道を着実に進んでいる」

編集部コメント

「会話型AI」という枠組みは、ここで一つの終わりを迎えるのかもしれない。Claudeが自分でマウスをクリックし、ボタンを押し、ダイアログを閉じるようになった今、AIとの関係は「質問と回答」から「指示と実行」へと本質的にシフトしたのだ。ビジネス領域では、ルーチンワークの大幅な削減が期待される。開発現場では、テストと検証の自動化が飛躍的に進むだろう。同時に、セキュリティと倫理の課題も一層重くなる。いかに信頼できるAIワーカーを育成し、その行動を適切に監視するか——その問い自体が、今後のAI産業を左右する最重要テーマになるに違いない。
