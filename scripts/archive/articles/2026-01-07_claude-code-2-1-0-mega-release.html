Claude Code 2.1.0 全貌解剖

2026年1月7日 ｜ Claude Code v2.1.0
2026年の幕開けを飾る大型アップデート「Claude Code 2.1.0」がリリースされた。1,096コミットを一気に投入し、Skills Hot Reload、Session Teleportation、コンテキストフォーク、多言語出力、ターミナルUX改善など、Claude Codeの使い方を根本から変える機能が満載だ。
1

Skills Hot Reload ── 5秒で反映される世界

~/.claude/skillsや.claude/skillsに置いたスキルファイルが、セッション再起動なしで即座に反映されるようになった。これまでスキルの修正→テストに2分かかっていたサイクルが、わずか5秒に短縮。

スキル開発のイテレーション速度が24倍に向上したとの報告もある。カスタムスキルの開発が一気に身近になった。

💡 スキルファイルを保存するだけで即反映。SKILL.mdの書き方を覚えれば、自分だけの専用AIツールを量産できる。
📎 VentureBeat報道を読む
2

Session Teleportation ── ターミナルからWebへ瞬間移動

/teleportコマンドで、ターミナルのセッションをclaude.ai/codeのWeb版にそのまま引き継げるようになった。自宅のターミナルで始めた作業を、外出先のブラウザで続行。会話履歴もコンテキストもすべて引き継がれる。

💡 /teleportでURLが発行される。そのURLを別デバイスで開くだけ。モバイルからでもセッションを引き継げる。
📎 Claude Code Changelog
3

Context Fork ── 安全な実験空間

スキルやスラッシュコマンドのfrontmatterにcontext: forkを指定することで、メインエージェントのコンテキストを汚さずにサブタスクを実行できるようになった。

新しいロジックのテストや、リスクのある操作を隔離環境で試せる。失敗してもメインセッションに影響しない安全設計だ。

📎 Claude Code Changelog
4

多言語出力 & ターミナルUX改善

language設定で出力言語を日本語やスペイン語などに指定可能に。日本語での開発ワークフローが格段にスムーズになる。

またShift+Enterによる改行がiTerm2、Kitty、Ghostty、WezTermで設定不要で動作するように。地味だが毎日使う機能だけに、この改善は大きい。

💡 設定ファイルに"language": "ja"と書くだけで、Claude Codeの出力が日本語になる。プロンプトごとに指定する手間がなくなった。
📎 Claude Code Changelog

Hooks強化 ── エージェントの振る舞いを制御

エージェント、スキル、スラッシュコマンドのfrontmatterにHooksを直接記述できるようになった。PreToolUse、PostToolUse、Stopのロジックをスキル単位でスコープできる。

これにより、スキルごとに異なるツール制約や監査ログを設定でき、きめ細かいエージェント制御が実現した。

💡 Hooksのスキル内定義は、チームでスキルを共有する際のガバナンスにも直結。「このスキルではBashの実行を禁止する」といった制約が宣言的に書ける。
📎 Claude Code Changelog
